1. Ringkasan Eksekutif

Saat ini, sistem CMS Bang Kesehatan menggunakan satu akun login yang ter-hardcode di dalam kode sumber. Dokumen ini merancang sistem User Directory lengkap dengan hierarki 4 peran (Superuser, Admin, Editor, Writer) dan kontrol akses berbasis peran (Role-Based Access Control / RBAC) yang dapat dikonfigurasi secara dinamis melalui antarmuka visual — tanpa perlu mengubah kode program.

✅ Pendekatan: Parameterized per Role Permission setiap role dapat diubah kapan saja oleh Superuser melalui halaman "Role Manager". Perubahan langsung berlaku untuk semua user yang memiliki role tersebut.

2.2 Deskripsi Peran

Peran Deskripsi Hak Akses
SUPERUSER Pemilik sistem. Hanya 1 orang. Tidak bisa dihapus. Semua fitur CMS tanpa batas. Mengelola User Directory, menambah/hapus user, mengubah role user lain, dan mengonfigurasi permission.
ADMIN Kepala Redaksi / Manager. Semua fitur konten (Dashboard, Editor, AI Generator, Analytics). Bisa menambah user Writer/Editor. Tidak bisa menghapus Superuser atau mengakses Role Manager.
EDITOR Penyunting/penulis senior. Bisa menggunakan Content Editor, AI Generator, dan meng-approve (Publish) artikel. Tidak bisa mengelola user.
WRITER Penulis/kontributor. Hanya bisa membuat artikel baru (Draft). Tidak bisa mempublish dan tidak bisa mengakses Analytics/User Directory.

3. Permission Matrix (Parameterized per Role)

Tabel di bawah ini adalah konfigurasi default yang akan ditanam saat pertama kali sistem dijalankan. Superuser dapat mengubah semua toggle ini kapan pun dari halaman Role Manager, kecuali kolom Superuser yang dikunci permanen.

Permission Key Label di UI Superuser Admin Editor Writer
dashboard Dashboard ✓ 🔒 ✓ ✓ ✓
editor Content Manager ✓ 🔒 ✓ ✓ ✓
ai_generator AI Generator ✓ 🔒 ✓ ✓ ✓
publish Publish Artikel ✓ 🔒 ✓ ✓ ✗
analytics Analytics ✓ 🔒 ✓ ✗ ✗
manage_users User Directory ✓ 🔒 ✓ ✗ ✗
manage_roles Role Manager ✓ 🔒 ✗ ✗ ✗
settings System Settings ✓ 🔒 ✗ ✗ ✗
ℹ️ Keterangan 🔒 = Dikunci permanen (Superuser selalu punya semua akses, tidak bisa diubah oleh siapapun).
✓ = Toggle ON (aktif).   ✗ = Toggle OFF (nonaktif).   Semua toggle selain kolom Superuser bisa diubah via halaman Role Manager.

7. Rencana Verifikasi

Build Test

# Memastikan tidak ada error kompilasi
npm run build

Manual Verification Checklist

  • Login sebagai Superuser → Semua menu muncul, bisa akses User Directory & Role Manager
  • Tambah user baru dengan role Writer → User muncul di tabel
  • Login sebagai Writer baru → Hanya 3 menu muncul (Dashboard, Editor, AI)
  • Writer ketik manual /cms/users di URL → Muncul halaman 403
  • Superuser buka Role Manager → Toggle "Analytics" untuk Writer → ON
  • Login ulang sebagai Writer → Menu Analytics sekarang muncul
  • Coba hapus akun Superuser → Sistem menolak
  • Nonaktifkan user → User tersebut tidak bisa login
